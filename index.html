Introduction to Cryptography

Core Concepts

  • Protects information from unauthorized access
  • Converts readable data (plaintext) into encrypted data (ciphertext)

Two Main Types

  • Symmetric Encryption: Uses a single key for encryption and decryption.
  • Asymmetric Encryption: Uses a key pair (Public & Private keys).

Main Technologies

Hash Functions (SHA)

  • Secure Hash Algorithm
  • Creates fixed-length digests
  • Protects data integrity

RSA Encryption

  • Asymmetric cryptography
  • Public and private key pairs
  • Used for digital signatures

SHA and Hash Functions

Key Features

  • SHA: Secure Hash Algorithm
  • Converts data into a fixed-length hash
  • One-way function (cannot be reversed)
  • Small input change → completely different hash
Input: Hello
Hash: 185f8db32271fe25f561a6fc938b2e264306ec304eba518007d1764826381969

Role in Blockchain

  • Each block contains the previous block's hash
  • Prevents modification of historically stored data
  • Creates complete chain integrity

RSA Algorithm

How It Works

RSA is an asymmetric encryption algorithm utilizing two distinct keys:

  • Public Key: Distributed openly; used to encrypt data / verify signatures.
  • Private Key: Kept strictly secret; used to decrypt data / create signatures.

Application In Blockchain

  • Used extensively for Digital Signatures
  • Verifies transaction ownership securely
  • Ensures robust authentication across nodes
RSA represents the concept of asymmetric cryptography in blockchain, though modern systems like Bitcoin often use ECC (Elliptic Curve Cryptography) for higher efficiency.

Practical Example: Bitcoin Transaction Verification

Transaction Flow

  • The sender creates a transaction and signs it digitally with a private key.
  • Nodes verify the signature using the corresponding public key.
  • Once validated, the transaction is added to a block with its hash.

Verification Summary

In Bitcoin, a transaction is first authenticated through digital signature verification, then it is hashed and included in the block structure to preserve integrity and traceability.

How Blockchain Uses Cryptography

A step-by-step breakdown of cryptographic integration within a block lifetime:

1

User creates a new transaction

2

Transaction is digitally signed using the user's RSA Private Key

3

Data is compiled and hashed using SHA Algorithm

4

The new block is linked securely to the previous block's hash

5

The entire decentralized network verifies the block validity
